### 一、已有研究方向

你的主线方向已经比较清晰，核心是：

**1. 大模型推理可靠性 / 容错计算**
你在简历中明确写的是“容错计算”，而论文和项目也都围绕 **LLM inference under soft errors / hardware faults** 展开，说明你的研究不是泛泛做大模型系统，而是聚焦在 **大模型推理阶段的可靠性保障**。 

**2. 面向 LLM 的软错误防护与鲁棒性增强**
论文 STAC 关注的是在高故障率硬件环境下，如何通过轻量机制提升 LLM 推理鲁棒性，核心问题是 **soft error / transient bit flip 导致的 silent data corruption**。这说明你已进入 **AI 系统可靠性** 这个较细分且有门槛的方向。 

**3. LLM 推理系统与执行机制**
你的工作不是只停留在算法层，而是强依赖对 **Prefill / Decode 两阶段、KV Cache、vLLM、PagedAttention、Continuous Batching** 等推理机制的理解，因此你的方向也可表述为：
**大模型推理系统 × 容错可靠性优化**。 

**4. 故障注入与可靠性评测基础设施**
你不仅做防护方法，还做了 Eval-FI 这类基础设施，说明你也有 **评测平台/实验系统构建** 的经历，研究方向可进一步概括为：
**LLM reliability evaluation、fault injection、causal analysis**。

### 二、已有研究经历

### 1. STAC 论文工作

这是你目前最核心、最完整的一段科研经历。

你以第二作者参与了论文 **STAC: Spatial-Temporal Activation Contextualization for Resilient LLM Inference**，投稿/发表目标为 DAC，主题是面向大模型推理容错。简历中写明，你针对 **Prefill / Decode 两类不同计算模式** 设计了差异化保护机制：

* Prefill 阶段，利用激活分布特征和通道 outlier 进行静态范围约束；
* Decode 阶段，针对 attention V 等脆弱模块，用 **XGBoost 回归树** 做动态范围预测与限制；
* 最终形成 **PD 分离的轻量保护框架**，在较低性能损失下显著提升容错能力。

论文正文进一步说明了这项工作的研究深度：
你们先提出并刻画了 **Outlier Dichotomy**，指出 LLM 推理中 Prefill 和 Decode 的激活行为具有根本不同的空间/时间特征，导致传统静态阈值防护失效；随后提出 **STAC**，分别用 Spatial Contextualizer 和 Temporal Contextualizer 去适配这两阶段的规律。 

从成果看，这项工作体现出你已经具备：

* 提炼系统问题的能力；
* 从机制分析到方法设计的完整科研链条；
* 将 ML 方法（XGBoost）嵌入系统可靠性问题中的能力；
* 做端到端实验、消融、开销分析的科研训练。
